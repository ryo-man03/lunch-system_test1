## 🎓 IQ110レベルでの技術解説

### 🔍 なぜ `orderId` を削除したのか？

**問題**:

```
注文に3つのID（orderId, orderCode, intakeCode）が混在
→ どれが「正しいID」なのか不明確
→ バグの原因になりやすい
```

**解決策**:

```
orderCode を唯一の主キーとして統一
intakeCode は後方互換性のため保持（orderCodeと同じ値）
→ シンプルで明確なデータ構造
```

**比喩**: 家の鍵が3本あるが、実際には1本しか使われていない状況。不要な2本を処分してシンプルにした。

---

### 🔄 なぜ `intakeCode` は残したのか？

**理由**: **後方互換性**

**シナリオ**:

```
古いスマホアプリ（旧バージョン）が intakeCode を期待している
→ orderCode だけを返すと、旧アプリが動かなくなる
→ intakeCode も返すことで、新旧両対応
```

**実装**:

```json
// サーバーのレスポンス
{
  "orderCode": "R-260216-Wzq",  // 新アプリ用
  "intakeCode": "R-260216-Wzq", // 旧アプリ用（同じ値）
  "order": {...}
}
```

**比喩**: 新しい住所に引っ越したが、旧住所宛の郵便も転送できるようにしている。
