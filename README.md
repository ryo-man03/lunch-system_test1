# 🍱 大学弁当予約システム

Node.js 標準ライブラリのみで構築された軽量な弁当予約管理システムです。
データ永続化、画像アップロード、キャンセル機能を備えています。

## ✨ 主要機能

### ユーザー機能

- 📱 メニュー閲覧・注文
- 🖼️ メニュー画像表示
- 🛒 カート機能（数量変更、削除）
- 📅 受取日選択（今日～7日先まで）
- 🔍 メニューフィルター（値段順、新着順、店舗ごと）
- 📋 受付番号コピー機能
- 🎟️ クーポン適用機能
- ❌ 注文キャンセル（条件付き）
- 📸 受付番号スクリーンショット促進

### 管理者機能

- 💳 決済処理（現金、PayPay、LINE Pay等6種類）
- 📦 受取管理
- 🍱 メニュー管理（画像アップロード、説明文、価格設定）
- 🏪 店舗管理（店舗情報、画像、連絡先）
- 📝 注文内容の変更・削除
- 📊 統計分析（売上、人気メニュー）
- 👥 管理者承認
- 📋 営業日・営業時間設定
- 💬 システムメッセージ設定

### システム機能

- 💾 **データ永続化** - JSON ファイル保存
- 🔄 **自動バックアップ** - 日次バックアップ（30 日間保持）
- 🖼️ **画像最適化** - 自動リサイズ（最大 800px、85%品質）
- 🔒 **キャンセルポリシー** - 注文後 30 分以内、または受取 24 時間前まで

## 👥 使用方法

### 学生として注文

1. 「学生として注文」を選択
2. **ステップ1**: 受取日を選択（今日～7日先まで）
3. **ステップ2**: メニューをフィルターで並び替え（値段順、新着順、店舗ごと）
4. メニューを選択してカートに追加
5. カート内で数量を変更（+/-ボタン）
6. クーポンコードがあれば入力
7. 「注文を確定」で受付番号を取得
8. 受付番号画面をスクリーンショット（大きく促進表示）
9. 窓口で受付番号を提示して決済

**キャンセル方法:**

- 注文後 30 分以内、または受取 24 時間前まで可能
- 「注文をキャンセル」ボタンから受付番号を入力

### 管理者としてログイン

**デフォルト管理者:**

- メール: `admin@example.com`
- パスワード: `admin123`

**管理者機能:**

1. **窓口業務**: 受付番号で決済処理、注文番号発行
   - 6 種類の決済方法選択（現金、PayPay、LINE Pay、メルペイ、クレジット、その他）
   - QR コード決済対応

2. **受取業務**: 注文番号で受取確認

3. **メニュー管理**:
   - 新規追加（JPEG/PNG 画像アップロード対応、自動最適化）
   - 価格・説明文の編集
   - 有効/無効の切り替え

4. **統計分析**: 売上、人気メニュー、決済方法の分析

## 🛠️ メンテナンス

### システム健全性チェック

```bash
node health-check.js
```

**チェック内容:**

- データファイルの整合性
- 統計情報（管理者、店舗、メニュー、受付、注文数）
- 未決済の受付リスト
- 古いデータの検出（30 日以上）
- バックアップ状況
- データファイルサイズ
- 画像データサイズ
- エラーログ

### データバックアップ

**自動バックアップ:**

- 毎日最初のデータ保存時に自動実行
- 保存先: `data/backups/backup-YYYY-MM-DD.json`
- 30 日以上前のバックアップは自動削除

**手動バックアップ:**

```bash
# データファイルをコピー
copy data\database.json data\backups\manual-backup.json
```

**データ復元:**

```bash
# バックアップファイルを復元
copy data\backups\backup-2025-01-08.json data\database.json
```

### データクリーンアップ

古いデータ（30 日以上前）が蓄積した場合:

```javascript
// health-check.jsで古いデータを確認
node health-check.js

// 手動でデータファイルを編集
// data/database.json の intakes, orders を整理
```

## 🔧 カスタマイズ

### ポート変更

`run_server_only.js` の `PORT` 変数を変更:

```javascript
const PORT = 8080; // お好みのポートに変更
```

### 画像サイズ制限

`public/js/admin.js` の `uploadImage` 関数:

```javascript
const maxSize = 5 * 1024 * 1024; // 5MB制限
const maxWidth = 800; // 最大幅800px
```

### キャンセルポリシー

`data/database.json` の `cancel_policy`:

```json
{
  "cancel_policy": {
    "enabled": true,
    "timeLimit": 30, // 注文後30分以内
    "beforePickupHours": 24 // 受取24時間前まで
  }
}
```

## 📝 API 仕様

### ユーザー API

- `GET /api/menu` - メニュー一覧
- `GET /api/stores` - 店舗一覧
- `POST /api/intake` - 注文受付
- `GET /api/intake/{code}` - 受付検索
- `POST /api/intake/{code}/cancel` - 注文キャンセル

### 管理者 API（認証必要）

- `POST /api/login` - ログイン
- `POST /api/logout` - ログアウト
- `POST /api/payment` - 決済処理
- `POST /api/pickup/{code}` - 受取処理
- `POST /api/menu/add` - メニュー追加
- `PUT /api/menu/{id}` - メニュー更新
- `DELETE /api/menu/{id}` - メニュー削除
- `PATCH /api/menu/{id}/toggle` - メニュー有効/無効切り替え
- `POST /api/stores/add` - 店舗追加
- `PUT /api/stores/{id}` - 店舗更新
- `DELETE /api/stores/{id}` - 店舗削除

## 🐛 トラブルシューティング

### データファイルが破損した場合

```bash
# バックアップから復元
copy data\backups\backup-YYYY-MM-DD.json data\database.json

# または、データファイルを削除して初期化
del data\database.json
node run_server_only.js
```

### 画像が表示されない場合

- 画像サイズが 5MB を超えていないか確認
- JPEG/PNG 形式か確認
- ブラウザのコンソールでエラーを確認

### パフォーマンスが遅い場合

```bash
# システム健全性チェックで確認
node health-check.js

# データファイルサイズが大きい場合はクリーンアップ
# 古い注文データを削除または別ファイルに移動
```

## 📊 パフォーマンス

- データファイルサイズ推奨: < 5MB
- 画像データ推奨: < 10MB
- 同時接続数: ~100 名（メモリ内データストア）
