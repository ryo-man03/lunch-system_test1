## システム起動・停止

### 起動方法

#### クイック起動（推奨）

```powershell
.\起動ファイル\クイック起動.ps1
```

または手動起動:

```powershell
cd c:\お弁当\lunch-system
node run_server_only.js
```

#### 起動確認

- ターミナルに以下が表示されれば成功:

```
🚀 サーバー起動: http://localhost:8080
📦 データファイル: data/database.json
✅ 準備完了
```

- ブラウザで http://localhost:8080 にアクセス

### 停止方法

- ターミナルで `Ctrl + C`
- または別のPowerShellで: `.\起動ファイル\すべて停止.ps1`

## 日常運用

### 📅 朝の準備（営業開始前）

#### 1. システム起動

```cmd
python simple-server.py
```

#### 2. システムヘルスチェック（オプション）

```powershell
node health-check.js
```

または管理画面で:

1. ブラウザで http://localhost:8080 を開く
2. 「管理者としてログイン」をクリック
3. Email: `admin@example.com`、Password: `admin123` でログイン
4. ダッシュボードで統計情報を確認

#### 3. 営業設定確認

1. 「⚙️ システム設定」をクリック
2. 営業時間が正しいか確認
3. システムメッセージが適切か確認

### 🍱 営業中の業務

#### 窓口業務（決済処理）

1. ダッシュボードから「📝 窓口業務」をクリック
2. 学生から受付番号（R-XXXX）を聞く
3. 受付番号を入力して「検索」
4. 注文内容を確認
5. 「💳 決済処理」ボタンをクリック
6. 発行された注文番号（4 桁英数字）を学生に伝える
7. **重要**: 学生に「この番号を受取時に必ず伝えてください」と案内

#### 受取業務

1. ダッシュボードから「📦 受取業務」をクリック
2. 学生から注文番号（4 桁英数字）を聞く
3. 注文番号を入力して「検索」
4. 注文内容と学生を確認
5. 弁当を渡す
6. 「✅ 受取完了」ボタンをクリック

#### エラー確認（随時）

1. ダッシュボードから「🔍 エラーログ」をクリック
2. エラーが表示されていないか確認
3. エラーがある場合は [エラー対応](#エラー対応) を参照

### 🌙 営業終了後

#### 1. 売上レポート確認

1. ダッシュボードから「📊 売上レポート」をクリック
2. 期間を「本日」に設定
3. 「📊 レポート生成」をクリック
4. 必要に応じて「🖨️ 印刷」

#### 2. ログ確認

1. ダッシュボードから「📋 ログ閲覧」をクリック
2. 本日の操作に問題がなかったか確認

#### 3. エラーログエクスポート（推奨）

1. 「🔍 エラーログ」をクリック
2. 「📥 エクスポート」ボタンで JSON ファイルをダウンロード
3. 日付をつけて保存（例: `error-log-2025-12-08.json`）

## トラブルシューティング

### 🔴 よくある問題と解決方法

#### 問題 1: サーバーが起動しない

**現象**: `Error: listen EADDRINUSE: address already in use :::8080`

**原因**: 既にサーバーが起動している

**解決方法**:

```powershell
# すべてのサーバーを停止
.\起動ファイル\すべて停止.ps1

# または手動で停止
Get-Process -Name node | Stop-Process -Force
```

#### 問題 2: 学生が受付番号を忘れた

**対応方法**:

1. 「📋 ログ閲覧」→「注文ログのみ」をクリック
2. 受付時刻・金額から該当の注文を探す
3. 受付番号（R-XXXX）を確認して伝える

#### 問題 3: 決済処理ができない

**原因と対応**:

| エラーメッセージ             | 原因         | 対応                 |
| ---------------------------- | ------------ | -------------------- |
| 「認証が必要です」           | ログイン切れ | 再ログイン           |
| 「受付番号が見つかりません」 | 番号間違い   | 受付番号を確認       |
| 「既に決済済みです」         | 二重決済     | 注文番号を学生に案内 |

#### 問題 4: ページが表示されない

**確認事項**:

1. サーバーが起動しているか確認
2. URL が `http://localhost:8080` になっているか確認
3. ブラウザのキャッシュをクリア（Ctrl + Shift + R）

#### 問題 5: 注文がキャンセルできない

**確認事項**:

- 注文後 30 分以内か？
- 受取 24 時間前より前か？
- 既に決済済みではないか？

**これらの条件を満たせばキャンセル可能**

## エラー対応

### 🔍 エラーログの見方

#### エラーログ画面

1. ダッシュボード → 「🔍 エラーログ」
2. 各エラーに以下の情報が表示されます:

```
[エラータイプ] JavaScript Error / API Error
[メッセージ] エラーの内容
[コンテキスト] エラーが発生した場所
[タイムスタンプ] 発生日時
[スタックトレース] 詳細情報（クリックで展開）
```

### 🚨 エラーレベルと対応

#### 🟢 軽微（無視可能）

- `ConnectionAbortedError`: ページ更新時の接続切断
- `304 Not Modified`: キャッシュ使用（正常）

**対応**: 不要

#### 🟡 警告（監視必要）

- API 応答遅延
- データ取得失敗（再試行で成功）

**対応**:

1. エラーログをエクスポート
2. 頻度を監視
3. 繰り返す場合は管理者に連絡

#### 🔴 重大（即対応）

- ログイン失敗
- データ保存エラー
- システムテスト失敗

**対応**:

1. サーバー再起動
2. ブラウザキャッシュクリア
3. それでも解決しない場合は技術サポートに連絡

### 📞 エスカレーション

#### 技術サポートへの報告内容

1. **エラーログのエクスポート**: JSON ファイルを添付
2. **発生状況**:
   - いつ: 日時
   - 誰が: 管理者/学生
   - 何を: 実行した操作
   - 結果: エラーメッセージ
3. **試したこと**: 再起動、キャッシュクリアなど

## メンテナンス

### 週次メンテナンス

#### データクリーンアップ（毎週月曜）

1. ログイン後、「🛠️ 詳細設定」をクリック
2. 「自動削除設定」セクションまでスクロール
3. 削除日数を確認（推奨: 30 日）
4. 「🗑️ 今すぐ実行」をクリック
5. 削除件数を確認

#### エラーログクリア

1. 「🔍 エラーログ」をクリック
2. 「📥 エクスポート」でバックアップ
3. 「🗑️ クリア」で削除

### 月次メンテナンス

#### 売上データのバックアップ

1. 「📊 売上レポート」をクリック
2. 前月全体のレポートを生成
3. 「🖨️ 印刷」で PDF 保存

#### メニュー・店舗の見直し

1. 「🍱 メニュー管理」で売上の少ないメニューを確認
2. 「🏪 店舗管理」で店舗情報を更新

### 学期末メンテナンス

#### データアーカイブ（新学期開始時）

1. 現在のデータベースをバックアップ

```powershell
Copy-Item "data\database.json" "data\archive\database-2026-spring.json"
```

2. 必要に応じてデータをクリーンアップ
   - 管理画面から古い注文データを確認
   - 売上レポートをPDF保存
   - 新学期用に設定を調整

**注意**: データは自動保存されるため、不要なデータは定期的に整理してください

## 🆘 緊急時対応

### システム停止時

1. **状況確認**: エラーメッセージを記録
2. **サーバー再起動**: `Ctrl + C` → `python simple-server.py`
3. **システムテスト**: 正常動作を確認
4. **営業再開**: 学生に案内

### データ消失の心配なし

**データベースの永続化**:

- すべてのデータは `data/database.json` に自動保存
- サーバー再起動後もデータは保持されます
- 日次で自動バックアップ（30日間保持）

**バックアップから復元する場合**:

```powershell
# 特定の日付のバックアップから復元
Copy-Item "data\backups\backup-2026-02-10.json" "data\database.json" -Force

# サーバー再起動
node run_server_only.js
```

### 大量注文発生時

1. 「📋 ログ閲覧」で受付状況を確認
2. 「🔔 店舗通知」で店舗ごとの注文数を確認
3. 各店舗に印刷して配布
4. 処理能力を超える場合は受付制限を検討
