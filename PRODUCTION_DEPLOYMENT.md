## リリース概要

### 🎯 目的

フロントエンド（user.js, admin.js, main.js）を旧API（intake系）から新API（order系）へ完全移行

### 📦 変更内容

- **影響範囲**: フロントエンドのみ（JavaScriptファイル3つ）
- **サーバー再起動**: 不要
- **データベース変更**: なし
- **ダウンタイム**: なし
- **後方互換性**: ✅ 維持（旧APIも引き続き動作）

### 🔒 リスク評価

| 項目           | リスクレベル | 理由                                   |
| -------------- | ------------ | -------------------------------------- |
| ダウンタイム   | 🟢 なし      | フロントエンドのみの変更               |
| データ損失     | 🟢 なし      | データベース変更なし                   |
| 機能停止       | 🟢 低        | 後方互換性あり、即座にロールバック可能 |
| パフォーマンス | 🟢 影響なし  | 処理ロジック変更なし                   |

## デプロイ手順

### 推奨デプロイ時間帯

- **🟢 最適**: 深夜0時〜5時（利用者少）
- **🟡 可能**: 10時〜11時、14時〜15時（昼・夕ピーク外）
- **🔴 避ける**: 11時〜13時、17時〜19時（注文ピーク）

### Step 1: ファイルのアップロード

#### 方法A: SCP/FTPで直接アップロード

```bash
# ローカルからサーバーへ
scp public/js/user.js  user@server:/path/to/app/public/js/
scp public/js/admin.js user@server:/path/to/app/public/js/
scp public/js/main.js  user@server:/path/to/app/public/js/
```

#### 方法B: Gitを使用（推奨）

```bash
# サーバー上で
cd /path/to/app
git pull origin main  # または該当ブランチ

# ファイル更新確認
git log --oneline -n 5
git diff HEAD~1 HEAD -- public/js/user.js
```

### Step 2: ファイル権限確認

```bash
chmod 644 public/js/user.js
chmod 644 public/js/admin.js
chmod 644 public/js/main.js

# 確認
ls -l public/js/*.js
```

### Step 3: キャッシュクリア

#### ブラウザキャッシュ

ユーザーに以下を案内（必要に応じて）：

- Windows: `Ctrl + F5`
- Mac: `Cmd + Shift + R`

#### CDN使用時

```bash
# 例: CloudFlare
curl -X POST "https://api.cloudflare.com/client/v4/zones/{zone_id}/purge_cache" \
     -H "Authorization: Bearer {api_token}" \
     -H "Content-Type: application/json" \
     --data '{"files":["https://yourdomain.com/public/js/user.js", ...]}'
```

#### サーバーサイドキャッシュ（該当する場合）

```bash
# Redis使用時
redis-cli FLUSHDB

# memcached使用時
echo "flush_all" | nc localhost 11211
```

### Step 4: 即座動作確認（クイックチェック）

```bash
# JavaScriptファイルの構文チェック（Node.js使用）
node -c public/js/user.js
node -c public/js/admin.js
node -c public/js/main.js

# 正常な場合は何も出力されない
# エラーがある場合は即座にロールバック
```

## トラブルシューティング

### 問題1: 「注文番号が見つかりません」エラー

**症状**: 正しい注文番号を入力しても見つからない

**原因**:

- データベースとフロントエンドの不整合
- 大文字小文字の違い

**対処法**:

```javascript
// ブラウザコンソールで確認
console.log('検索コード:', document.getElementById('order-code').value);

// サーバーログ確認
tail -f /var/log/lunch-system/server.log
```

### 問題2: JavaScriptエラー「Cannot read property 'orderCode' of undefined」

**症状**: ブラウザコンソールにエラー表示

**原因**:

- APIレスポンスの構造が期待と異なる
- 古いキャッシュが残っている

**対処法**:

```bash
# 1. ブラウザキャッシュ完全クリア
Ctrl + Shift + Delete → すべてクリア

# 2. サーバーサイドキャッシュクリア
# (該当する場合)

# 3. それでも解決しない場合はロールバック
```

### 問題3: 決済処理で「認証が必要です」エラー

**症状**: 管理者画面で決済処理が失敗する

**原因**:

- セッションが無効
- Cookie設定の問題

**対処法**:

```bash
# 1. 再ログイン
# 管理者画面でログアウト → 再ログイン

# 2. Cookieの確認
# ブラウザのDevTools → Application → Cookies
# sessionId の存在確認

# 3. サーバーログ確認
grep "認証" /var/log/lunch-system/server.log
```

### 問題4: 画面が真っ白

**症状**: ページにアクセスすると白い画面

**原因**:

- JavaScriptの構文エラー
- ファイルが破損

**対処法**:

```bash
# 1. ブラウザコンソール確認（F12）
# エラーメッセージを確認

# 2. ファイル構文チェック
node -c public/js/user.js
node -c public/js/admin.js
node -c public/js/main.js

# 3. 即座にロールバック
```
